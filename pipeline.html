LIDC-IDRI 전처리 파이프라인

2.5D 슬라이스 기반 폐 결절 분류 연구 — 코드 문서 & 진행 현황
연구 가설
2.5D 슬라이스 입력 방식(중심 슬라이스 ± N장)으로 결절 주변 맥락을 포함하여 학습하면, 단일 슬라이스 2D CNN 대비 AUC를 비롯한 분류 성능이 향상된다.
전제: Z방향 맥락 정보(결절의 3D 형태, 주변 조직 패턴)가 악성도 판별에 유의미한 feature를 제공한다.

parse_lidc_annotations.py
역할 LIDC XML annotation 파싱 → 환자별 결절 딕셔너리 구성 → JSON 저장 (1단계)
주요 함수 4개
build_uid_map() — metadata.csv → {series_uid: {subject_id, file_location}} O(1) 조회 맵. 루프마다 CSV 탐색 방지.

collect_raw_nodules() — ReadingSession 순회, malignancy 1-5 범위 외 제외, polygon ROI 추출. 핵심 처리: z_raw 이상값(|z|>400mm) 자동 감지 → deci-mm 의심 시 /10 보정 + z_suspicious 플래그. MIN_POLY_PTS 미만 polygon 제외.

match_nodules() — O(n²) greedy 매칭. 조건: XY 유클리드 ≤ NODULE_XY_THR(5px) AND |Δz| ≤ NODULE_Z_THR(8mm). 같은 reader_id의 중복 병합 금지 로직 포함.

compute_derived() — all_z_positions(정렬), center_z(판독자 최다 동의 z, 동률 시 median), rep_polygon(center_z에서 꼭짓점 수 최다), center_x/y 계산. bbox_2d/3d는 주석 처리(팀원과 결정 대기).
match_dicom.py
역할 file_location → DICOM 직접 접근 → CT 공간 메타 추출 → 결절 물리 직경 계산 (이전 버전 대비 SeriesUID 전수 탐색 제거, 속도 향상)
주요 함수
resolve_dicom_dir() — SERVER_ROOT + file_location('./' 제거) → 절대경로 변환. 폴더 존재 검사.

extract_ct_meta() — stop_before_pixels=True로 헤더만 읽어 빠름(~수ms/파일). 추출 항목: rows/cols/num_slices, pixel_spacing_x/y, z_spacing, rescale_slope/intercept, bits_stored, window_center/width. 제조사/커널은 주석 처리.

compute_nodule_diameter_3d() — polygon vertex를 mm 공간(× pixel_spacing) 변환 → x/y/z 범위 → diameter_max_mm. nodule_info_clean.json의 derived에 추가.

plot_dicom_histograms() — 883 시리즈의 rows/cols/슬라이스수/spacing 4종 히스토그램 생성 (dicom_histogram.png).
export_nifti.py
역할 DICOM → 1mm 등방성 NIfTI 변환, polygon annotation → 3D segmentation mask 렌더링
핵심 설계: 반전 DICOM 처리 (LIDC-IDRI-0085 케이스)
문제: SimpleITK GetSpacing()[2]은 항상 양수 반환 → 내림차순(반전) DICOM에서 z_vox 음수 계산 오류
해결: SliceLocation 태그(0020|0032) 직접 읽어 actual_step = sl_1 - sl_0 (음수 가능) 계산
공식: z_vox = round((z_mm - sl_origin) / actual_step)
normalize_hu() — clip(HU_MIN=-1000, HU_MAX=400) → 선형 [0,1]. ct_norm.nii.gz 저장.

build_seg_mask() — skimage.draw.polygon으로 2D polygon을 각 슬라이스에 렌더링 → 3D mask. actual_step 부호로 반전 케이스 자동 처리. 값 = nodule_idx+1.

process_patient() — 복수 series 처리, center_z 기준 nodule→series 배정, 범위 밖은 최근접 series에 강제 배정 + WARN 로그.
validate_grouping.py
역할 매칭 임계값(5px, 8mm)의 데이터 기반 근거 확보 + sanity check
• 판독자 간 centroid 거리 분포 히스토그램 → 95th percentile 기반 임계값 정당화 (centroid_distribution.png)
• group_size > 4 케이스 탐지 (4명 초과 시 오매칭 의심)
• 결절 없는 series 원인 분석, DICOM spacing 분포 확인
• config.py의 NODULE_XY_THR / NODULE_Z_THR을 직접 import → 임계값 변경 시 자동 반영

즉시 실행 — 오늘 / 이번 주
STEP A 긴급
seg nonzero 전수 검사
export_nifti.py 상단 주석의 검증 스크립트 실행 → 빈 mask 환자 목록 파악 → 실제 유효 결절 수 확정. 이 수치가 2.5D 실험의 기준 결절 수가 됨.
STEP B 긴급
결절 패치 추출 구현
nodule_info_clean.json의 center_x/y/z 좌표로 ct_norm.nii.gz에서 64×64 ROI crop 함수 구현. bbox_2d 주석 해제 여부 결정. 경계 처리(패딩) 로직 포함.
STEP C 긴급
2.5D Dataset 클래스 수정
현재 단일 슬라이스 Dataset을 center_z ± N 슬라이스 스택 방식으로 수정. config 파라미터로 N 전환 가능하게 설계. WeightedRandomSampler 추가(악성 클래스 불균형 보정).
STEP D
베이스라인 2D 학습 → AUC 측정
베이스라인 3-conv CNN으로 학습 완료 후 Test set AUC 측정. 이 수치가 2.5D 비교 기준선. Weighted cross-entropy 또는 focal loss 적용. Random flip/rotation augmentation 추가.
STEP E
2.5D CNN 구현 + 비교 실험
입력 채널 수를 config로 받는 2.5D 모델 구현. N=0(2D, 1ch), N=1(3ch), N=2(5ch), N=3(7ch) 동일 조건(split/seed/lr/epoch)으로 실험. 결과 표: AUC / macro-F1 / 악성 recall 비교.
